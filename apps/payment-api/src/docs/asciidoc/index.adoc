== 공통 헤더

모든 엔드포인트는 아래 헤더를 필요로 합니다.

[cols="1,1,3"]
|===
| 헤더 | 필수 | 설명

| `Idempotency-Key`
| Y
| 멱등성 키. UUID 또는 고유 문자열. 동일 키로 재요청 시 이전 결과를 그대로 반환합니다.

| `X-User-Id`
| Y
| 인증된 사용자 ID (Long).
|===

== 결제 API

잔액을 차감해 가맹점에 결제합니다. 동시성 제어 전략별로 엔드포인트가 분리되어 있습니다.

[cols="1,3"]
|===
| 엔드포인트 | 동시성 전략

| `POST /payments/pessimistic`
| 비관적 락 (`SELECT ... FOR UPDATE`) — 충돌이 잦을 때 안정적

| `POST /payments/optimistic`
| 낙관적 락 (`@Version`) — 충돌이 적을 때 높은 처리량

| `POST /payments/redis-lock`
| Redis 분산 락 (Redisson) — 다중 인스턴스 환경에서 전역 직렬화

| `POST /payments/atomic`
| Atomic UPDATE (`UPDATE ... WHERE balance >= amount`) — 락 없이 단일 쿼리
|===
